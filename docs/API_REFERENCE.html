Auth — Xác thực người dùng

Quản lý đăng ký, đăng nhập và đăng xuất. Token được lưu phía client (localStorage). Mọi route yêu cầu xác thực phải gửi kèm Authorization: Bearer <token>.

PUBLICPostgreSQL
MethodEndpointQuyềnMô tả
POST /auth/register Public Đăng ký tài khoản mới. Body: { name, email, password }. Tạo tài khoản trong PostgreSQL và trả về UUID auth token.
POST /auth/login Public Đăng nhập bằng email và mật khẩu. Trả về UUID auth token và recommendationRefresh; login chỉ queue/promote background job.
POST /auth/logout Auth Đăng xuất, vô hiệu hóa token hiện tại của người dùng đang đăng nhập.

Properties — Danh sách bất động sản

API chính cho trang web công khai. Trả về danh sách BĐS đã phân tích và chuẩn hóa từ PostgreSQL. Hỗ trợ filter đa tiêu chí, sắp xếp và phân trang.

PUBLICPostgreSQL
MethodEndpointQuyềnMô tả
GET /properties Public Lấy danh sách BĐS với filter đa chiều: loại (nhà/đất/chung cư), giá, diện tích, tỉnh/thành, trạng thái (ACTIVE/SOLD...). Hỗ trợ sort và phân trang.
GET /properties/:id Public Lấy chi tiết một BĐS theo ID: địa chỉ, giá, diện tích, số phòng, mô tả, ảnh và nguồn gốc bài đăng thô.
POST /properties/me Auth Seller tao tin dang tu form co cau truc. Luu truc tiep vao PostgreSQL Property voi DRAFT, khong tao RawPost va khong auto-analysis.
GET /properties/me Auth Lay danh sach tin dang cua user hien tai, gom DRAFT, PENDING_REVIEW, PUBLISHED va ARCHIVED.
GET /properties/me/:id Auth Lay chi tiet tin dang thuoc user hien tai.
PATCH /properties/me/:id Auth Sua tin dang cua user khi con DRAFT hoac PENDING_REVIEW. Sua tin cho duyet se dua ve DRAFT.
PATCH /properties/me/:id/submit Auth Gui tin nhap sang trang thai PENDING_REVIEW de admin duyet.

Recommendations (User) — Gợi ý BĐS cá nhân hóa

Hệ thống gợi ý BĐS dựa trên nhu cầu và hành vi của người dùng. Sử dụng hybrid ranking: điểm khớp quy tắc + preference boost + feedback boost + freshness boost. Kết quả thuật toán lưu trong DemandPropertyMatch.

AUTH REQUIREDPostgreSQL
MethodEndpointQuyềnMô tả
GET /recommendations/me Auth Lấy danh sách BĐS được gợi ý cho người dùng, đã xếp hạng theo hybrid score (0–100). Filter theo status (ACTIVE / DISMISSED...) và phân trang.
POST /recommendations/matches/:matchId/feedback Auth Gửi phản hồi cho một gợi ý (LIKED / DISLIKED / CONTACTED). Phản hồi được dùng để điều chỉnh preference profile và cải thiện gợi ý tiếp theo (learning loop).
POST /recommendations/me/recompute Auth Kích hoạt tính toán lại toàn bộ gợi ý cho người dùng hiện tại dựa trên nhu cầu và profile mới nhất.

User Behaviors — Theo dõi hành vi người dùng

Thu thập và lưu sự kiện hành vi của người dùng trên website (xem BĐS, tìm kiếm, lọc, yêu thích, gửi yêu cầu tư vấn, xem gợi ý...). Dữ liệu này được dùng trong learning loop để cải thiện chất lượng gợi ý. Tracking chỉ chạy khi có token.

AUTH REQUIREDPostgreSQL
MethodEndpointQuyềnMô tả
POST /user-behaviors Auth Ghi nhận một sự kiện hành vi. Body: { eventType, propertyId?, metadata? }. EventType: PROPERTY_VIEW, SEARCH, FILTER_APPLY, FAVORITE_ADD, INQUIRY_SUBMIT, RECOMMENDATION_VIEW...
GET /user-behaviors/me Auth Xem lịch sử hành vi của người dùng đang đăng nhập. Filter theo eventType, khoảng thời gian và phân trang.

Lưu và tính toán preference profile được suy luận từ hành vi thực tế của người dùng. Profile này ảnh hưởng đến điểm số hybrid ranking của gợi ý BĐS. Chỉ lưu sở thích BĐS, không lưu dữ liệu nhạy cảm.

AUTH REQUIREDPostgreSQL
MethodEndpointQuyềnMô tả
GET /user-preferences/me Auth Lấy preference profile hiện tại: loại BĐS ưa thích, khoảng giá, diện tích, khu vực — được suy luận từ lịch sử hành vi.
POST /user-preferences/recompute Auth Tính toán lại toàn bộ preference profile từ lịch sử hành vi. Gọi sau khi có nhiều hành vi mới để cập nhật profile chính xác hơn.

Recommendations (Admin) — Quản lý hệ thống gợi ý

Admin chạy thuật toán khớp nối giữa nhu cầu người dùng và BĐS, xem kết quả match và cập nhật trạng thái.

ADMIN ONLYPostgreSQL
MethodEndpointQuyềnMô tả
POST /admin/recommendations/user-demands/:demandId/run Admin Chạy thuật toán rule-based matching (6 tiêu chí, thang 0–100) cho một UserDemand cụ thể. Tạo hoặc cập nhật các DemandPropertyMatch.
GET /admin/recommendations/user-demands/:demandId/matches Admin Xem danh sách BĐS đã khớp với một nhu cầu người dùng, sắp xếp theo hybrid score. Filter theo status: ACTIVE, DISMISSED, CONTACTED, OUTDATED.
PATCH /admin/recommendations/matches/:matchId/status Admin Admin cập nhật trạng thái của một match (ví dụ: OUTDATED nếu BĐS đã bán, CONTACTED nếu đã liên hệ người dùng).

GET /admin/source-imports/:id Admin Chi tiết một lô import: tổng records, số CREATED / SKIPPED / FAILED và kết quả xử lý từng record.

External Behaviors — Hành vi bên ngoài & Liên kết người dùng

Quản lý hành vi từ nguồn DEV hoặc social/partner đã được cấp quyền. Admin liên kết externalUserRef với user HomeSense, import behavior qua POST /admin/source-imports/json với target EXTERNAL_BEHAVIOR. Nguồn production phải có AUTHORIZED_API, PARTNER_AGREEMENT hoặc USER_SUBMITTED; quyền public đơn thuần không đủ để gắn hành vi cá nhân. PostgreSQL background job chạy rule-based/cache trước Gemini fallback (tối đa 25 records/64KB). LLM không tham gia ranking.

ADMIN ONLYPostgreSQLMongoDB
MethodEndpointQuyềnMô tả
GET /admin/external-behaviors Admin Danh sách bản ghi hành vi bên ngoài dạng raw từ nguồn được quản trị. Filter theo dataSourceId, externalUserRef, khoảng thời gian. Có phân trang.
GET /admin/external-behaviors/:id Admin Chi tiết một bản ghi hành vi bên ngoài theo MongoDB ObjectId. Bao gồm payload, externalUserRef và trạng thái phân tích.
POST /admin/external-user-links Admin Liên kết một externalUserRef từ nguồn DEV hoặc social/partner đủ quyền với user HomeSense. Body: { dataSourceId, externalUserRef, userId }. Mỗi cặp (dataSourceId, externalUserRef) là duy nhất.
GET /admin/external-user-links Admin Danh sách liên kết người dùng bên ngoài. Filter theo dataSourceId, userId, trạng thái. Có phân trang.
PATCH /admin/external-user-links/:id/status Admin Kích hoạt hoặc vô hiệu hóa liên kết người dùng bên ngoài. Body: { isActive: boolean }.
Import hành vi: POST /admin/source-imports/json hỗ trợ target EXTERNAL_BEHAVIOR với items { externalUserRef, payload, externalId?, occurredAt? }.
Background learning: Khi user đăng nhập, hệ thống chỉ queue/promote job. Worker xử lý hành vi bên ngoài, tạo ExternalBehaviorAnalysis, canonical signals/intents và recompute recommendation. Response login có thêm recommendationRefresh.
